## What is Sponsored Prompt Opener?
Sponsored Prompt is an ad format where a brand sponsors a suggested prompt in your chat interface.   
When a user clicks it, the chatbot responds normally with a brand message appended at the end.

<img src="/assets/sponsored-prompt-opener.png" style={{ width: '50%' }} />

When the user clicks the sponsored prompt:

1. Send the prompt text as the user's message to the LLM
2. Append the brand's `append_message` and CTA below the LLM response

<img src="/assets/sponsored-prompt-ad-render.png" style={{ width: '50%' }} />

---

## Request

Call the SSP with `ad_format` set to `"sponsored_prompt"`. No conversation messages are needed — this is a chat opener.

<Note>
  To test without a live auction, set `"production": false`. An ad is always returned in sandbox mode.
</Note>

```json
{
  "userId": "user_123",
  "ad_format": "sponsored_prompt",
  "country_code": "US",
  "production": false
}
```

---

## Response

The SSP returns 3–5 sponsored prompts to display as suggestions.

```json
{
  "requestId": "api_req_789",
  "status": "success",
  "data": {
    "bids": [
      {
        "bidId": "sp_bid_abc123",
        "advertiser": "Nike",
        "logo_url": "https://cdn.example.com/nike-logo.png",
        "image_url": "https://cdn.example.com/nike-shoes.jpg",
        "sponsored_prompt": "What are the best running shoes for marathons?",
        "append_message": "Check out the Nike Air Zoom Alphafly — the marathon record-holder's shoe.",
        "cta_text": "Shop Now",
        "cta_url": "https://ssp.thrads.ai/api/v1/tracking/redirect?token=...",
        "view_url": "https://localhost:8000/api/v1/tracking/view?token=...",
        "sponsored_prompt_click_url": "https://events.thrad.ai/api/v1/tracking/sponsored-prompt-click?token=...",
        "sponsored_prompt_ad_click_url": "https://events.thrad.ai/api/v1/tracking/sponsored-prompt-click?token=...",
        "placement": "sponsored_prompt"
      }
    ]
  }
}
```

| Field | Description |
|-------|-------------|
| `sponsored_prompt` | Text to display as the suggested prompt |
| `append_message` | Brand message to append after the LLM response |
| `cta_text` | Label for the CTA button |
| `cta_url` | CTA button destination (tracking redirect) |
| `logo_url` | Brand logo to display in the sponsored card |
| `image_url` | Product image (optional display) |
| `view_url` | Fire when the sponsored prompt becomes visible to the user |
| `sponsored_prompt_click_url` | Fire when the user taps the sponsored prompt text |
| `sponsored_prompt_ad_click_url` | Fire when the user taps the CTA button |

---

## No Bid

If no sponsored prompts are available, the response returns an empty bids array. Fall back to your default organic suggestions.

```json
{
  "data": {
    "bids": []
  }
}
```

## Tracking

Fire each URL as a GET request — no body needed.

<Note> In mobile apps, we track rendering events via URL calls.</Note>

| URL | When to fire |
|-----|-------------|
| `view_url` | Prompt card becomes visible on screen |
| `sponsored_prompt_click_url` | User taps the prompt text |
| `sponsored_prompt_ad_click_url` | User taps the CTA button |

```swift iOS (Swift)
// Minimal tracking helper — fire and forget
func fireTracking(_ urlString: String) {
    guard let url = URL(string: urlString) else { return }
    URLSession.shared.dataTask(with: url).resume()
}

// 1. Card is visible
fireTracking(bid.view_url)

// 2. User taps the sponsored prompt
fireTracking(bid.sponsored_prompt_click_url)
// → send bid.sponsored_prompt as the user's message to your LLM

// 3. User taps the CTA button
fireTracking(bid.sponsored_prompt_ad_click_url)
// → open the destination URL
UIApplication.shared.open(URL(string: bid.cta_url)!)
```
